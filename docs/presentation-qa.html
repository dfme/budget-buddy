Quality Attributes

Top 3 kritische Quality Attributes

# 1 · Kritisch
🔐
Sicherheit & Datenschutz
Lara und Marc übergeben ihr vollständiges Finanzbild. Marc's Haupthürde ist explizit Datenschutz-Skepsis. Das nDSG schreibt Recht auf Löschung vor. Ein Datenleck ist laut eigenem Risikocatalog existenzbedrohend.
⚠️ Risiko ohne QA: Datenleck → Vertrauensverlust + nDSG-Busse → Projektende
# 2 · Hoch
🖱️
Usability
Laras Hürde: „Wenn der erste PDF-Upload zu kompliziert ist, bricht sie sofort ab." Der Onboarding-Funnel ist der einzige Moment um sie zu gewinnen. Risiko #1 im Projekt ist die Churn-Falle.
⚠️ Risiko ohne QA: User bricht nach erstem Upload ab → Conversion = 0%
# 3 · Hoch
🎯
Korrektheit
Core Value: „A Safe-to-Spend number users can trust." User Stories fordern «auf den Rappen genau» an 5 Stellen. Falsche Zahlen treffen Laras Mietentscheidung — das ist kein Bug, das ist Vertrauensbruch.
⚠️ Risiko ohne QA: Falsche Safe-to-Spend → Lara zahlt Miete nicht → Vertrauen weg
Architektonische Massnahmen

Massnahmen pro Quality Attribute

🔐 Sicherheit & Datenschutz
bcrypt (Work Factor ≥ 12) Passwörter nie im Klartext — US-1 AC fordert es explizit
PDF nur im RAM parsen Kein transferTo() — nach Extraktion sofort verwerfen (US-4)
User-ID-Filter auf ALLEN Queries JPA Repository: User A sieht nie Daten von User B
Kaskadierendes DELETE ON DELETE CASCADE in Flyway — nDSG Recht auf Löschung (US-2)
JWT-Secret als Env-Variable ${JWT_SECRET} in application.yml — nie im Git-Repository
🖱️ Usability
Fortschrittsanzeige beim Upload Spinner + „42 Transaktionen erkannt" — US-4 fordert Erfolgsmeldung
Inline-Fehlermeldungen mit Kontext Nicht „Fehler" — „PDF ist passwortgeschützt, bitte Schutz entfernen"
Fixkosten-Wizard mit Escape Nicht überspringbar, aber „Keine Fixkosten"-Option vorhanden (US-3)
30s Timeout mit klarer Meldung Lara wartet nie unbegrenzt auf einen hängenden Upload
Automatische Einkommensschätzung Regelmässige Gutschrift ±5% erkannt → Vorschlag ohne manuelle Eingabe
🎯 Korrektheit
BigDecimal überall — kein double/float 0.1 + 0.2 ≠ 0.3 in double — DB als NUMERIC(15,2)
Apostrophe-Parsing für CHF-Format replace("'", "") vor BigDecimal-Parse — 1'234.56 ist kein valides Java
Saldo-Anker beim PDF-Parsing Mehrzeilige Textfelder korrekt trennen — Saldo-Spalte als Zeilenanker
Divisor mindestens 1 Safe-to-Spend am Monatsende: Division durch 0 explizit abgefangen (US-6)
Quartals-/Jahres-Normalisierung 300 CHF/Quartal → 100 CHF/Monat (÷3); Fehler hier bricht jede Berechnung
