10. Juni · Fundament gelegt

Zwei nicht verhandelbare Regeln

🔑 Keine Secrets im Git

Credentials, API-Keys & Passwörter nie ins Repo. .env in .gitignore; ANTHROPIC_API_KEY und JWT-Secret nur als Umgebungsvariablen — nie in Code oder application.properties.

Bei Leak: sofort Key-Rotation + nDSG-Incident-Assessment

💰 Geld immer als BigDecimal

Alle CHF-Beträge in Entities, Services, DTOs & Berechnungen als BigDecimal. double/float verboten (ADR-9: Rundungsfehler beim Safe-to-Spend).

+ DECIMAL(10,2) in der DB
+ replace("'", "") vor new BigDecimal(…)

Das Sicherheits- und Korrektheits-Fundament — Voraussetzung für alle späteren Conventions.

22. Juni · Karpathy-Bonus (2/2)

Guardrails: Was Claude nicht tun darf

Git & Workflow

Kein Direkt-Commit auf main, kein Self-Merge, keine Co-Authored-By, keine Emojis.

Geld & Berechnung

Nie double/float — immer BigDecimal. Apostroph-Trenner vor dem Parsen entfernen.

Sicherheit & Daten

Keine Secrets/PII, kein findById ohne Owner-Check, foreign_keys nie aus.

Tech-Stack-Grenzen

Kein Gradle, WebFlux, NgRx, H2-Tests, iText — kein Spring Boot 4.

Architektur

Kein modulübergreifender Direktzugriff, Claude nur hinter CategorizationPort, Timeout+Fallback.

Scope

Nichts ausserhalb Schweiz-MVP; ADRs nicht eigenmächtig überschreiben.

Meta-Regel: bei Konflikt stoppen und nachfragen — nicht umgehen.
